An In-Depth Guide to Reading stdin in C on Linux

By Linux Code / October 24, 2023

As Linux programmers, we often need to write C programs that interact with users via the terminal. This requires reading input that users provide via the standard input stream (stdin) and printing results to standard output (stdout). Mastering stdin is key for writing robust, interactive command-line programs in Linux.

In this comprehensive guide, you‘ll learn various methods for reading from stdin in C on Linux systems. I‘ll provide background on Linux streams, examples of correctly handling stdin, common pitfalls to avoid, and best practices for writing solid C programs that handle user input well. Let‘s dive in!

What is stdin and Why is it Important?

The stdin stream represents the standard input source for a Linux process. By default, this comes from the terminal, but can be redirected in various ways. For example:

  • A user types input at a terminal prompt
  • Input is piped from another process via |
  • stdin is redirected from a file via <
  • Input comes from /dev/null or /dev/zero

This flexibility makes stdin useful for:

  • Accepting user input in interactive C programs
  • Processing data piped between programs
  • Parsing input files line-by-line
  • Embedding default input data into programs

Here is an example pipeline sorting stdin lines and printing the sorted result:

$ cat myfile.txt | sort | ./myprog

And a C program that reads numbers from stdin:

int main() {
  int num;
  while (scanf("%d", &num)) {
    printf("num=%d\n", num); 
  }
}

Being able to handle stdin well is crucial for writing command-line utilities and Linux scripts in C.

Buffering of Stdin and Stdout

stdin and stdout are typically line buffered when connected to a terminal. This means output is flushed on newline, and input is available line-by-line.

But in some cases, stdin/stdout may be fully block buffered for performance. This buffers data in chunks before reading/writing.

So C programs should avoid assuming line buffering. Explicitly flush stdout before reading stdin, or disable buffering entirely with setbuf:

setbuf(stdout, NULL); // disable stdout buffering

Piping data between programs often enables block buffering as well. So when writing data processing programs in C, use line buffering or explicit fflush calls.

Reading Stdin in Bash vs C

Bash provides many easy ways to read from stdin, like:

  • read var – read line into variable
  • readarray arr – read lines into array
  • while read line; do ... – read line-by-line

But in C, stdin handling must be done manually via functions like fgets(), scanf(), etc.

Bash implicitly handles piping stdin between commands. But C programs need to explicitly check for EOF when reading from a pipe.

Let‘s look at some proper ways to handle stdin in C.

Reading Lines with fgets()

The fgets(str, size, stream) function allows reading a line of text from stdin:

char line[100];
fgets(line, sizeof(line), stdin);

Some tips for using fgets() correctly:

  • Check the return value – it‘s NULL on error/EOF
  • Pass a buffer size to prevent overflows
  • Handle lines longer than the buffer size
  • Use loops for reading multiple lines
  • Explicitly flush stdout before reading stdin

This example robustly reads multiple lines until EOF:

int main() {
  char line[100];

  while (fgets(line, sizeof(line), stdin)) {  
    // check for long lines

    printf("> %s", line); // flush stdout
    fflush(stdout); 
  }
  return 0; 
}

fgets() is the recommended way for reading full lines from stdin in C.

Dangers of gets()

The gets(str) function can also read a line from stdin and is simpler to use than fgets():

char line[100];
gets(line);

However, gets() is dangerous and should never be used because:

  • It does not limit input length, so buffer overflows are possible
  • No way to tell if EOF was reached or error occurred
  • Not part of the current C standard

So for safety, always use fgets() over gets() when reading stdin.

Reading Formatted Input with scanf()

The scanf() family of functions can read various types of formatted input from stdin:

int num;
float price; 

scanf("%d", &num); // read integer
scanf("%f", &price); // read float

scanf() is good for simple input, but not full lines. Watch out for these pitfalls:

  • No protection against long input
  • Limited error handling
  • Difficult to handle character data

Prefer line-oriented fgets() if reading arbitrary input from users. Use scanf() for reading structured data.

Clearing stdin Between Reads

Sometimes stdin can get in an error state after malformed input. This will cause future reads to fail instantly.

We can clear the error flag on stdin by calling fflush(stdin):

if (fgets(line, sizeof(line), stdin) == NULL) {
  fflush(stdin); // reset error state
}

This discards any residual bad data and resets stdin for more reading.

Read Lines Dynamically with getline()

The POSIX getline() function allows dynamically growing the buffer to fit each line read from stdin:

size_t len = 0;
char *line = NULL;

while (getline(&line, &len, stdin) != -1) {
  // line contains data
} 

free(line);

This is useful for reading unknown length input. But getline() is not in standard C – you‘d need to feature test for support.

Putting it All Together: Robust Input Handling

Here is an example program demonstrating robust stdin handling in C:

#include <stdio.h>

#define MAX_LINE 100

// Reads line from stdin, returns length
size_t read_line(char *buf, size_t size) {
  if (!fgets(buf, size, stdin)) {
    return -1; // error or EOF
  }

  // strip newline 
  buf[strcspn(buf, "\n")] = 0;

  return strlen(buf); 
}

int main() {
  char line[MAX_LINE];
  size_t len;

  printf("Enter some lines (EOF to quit):\n");

  while (1) {
    // flush stdout before read
    fflush(stdout);  

    len = read_line(line, MAX_LINE);
    if (len == -1) {
      break; // EOF
    } 

    printf("Read %zd chars: %s\n", len, line);
  }

  return 0;
}

This demonstrates multiple best practices like:

  • Checking return values from fgets()
  • Flushing stdout before reads
  • Handling long lines properly
  • Encapsulating stdin reads in a function
  • Breaking on EOF

Following these techniques will ensure your C programs interact with stdin reliably.

Recommended Practices for Production Code

When writing production programs that need to handle stdin properly, keep these best practices in mind:

  • Use fgets() over gets() for safety
  • Validate line length and truncate if too long
  • Flush stdout before reading stdin
  • Actively check for EOF rather than assume input
  • Avoid arbitrary limits on line lengths
  • Reset stdin state after errors with fflush(stdin)
  • Prefer portable functions like fgets() over POSIX ones
  • Disable buffering entirely for piped data
  • Read stdin in chunks/loops instead of all at once

Adopting defensive coding practices like these will prevent crashes, overflow issues, and weird pipe behavior in C applications that process stdin.

Conclusion

Stdin processing is a key skill for writing effective command-line programs and Unix utilities in C. Mastering fgets() usage, line-by-line reading, proper error handling, and stdout flushing will enable you to handle input from users, files, and pipes gracefully. This will open up many possibilities for powerful data processing programs.

I hope these examples and best practices provide a helpful overview of how to read from stdin robustly in C on Linux systems. Let me know if you have any other questions!
